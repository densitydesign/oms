PROTOCOL_01

harvesting the web corpus

We collected the first 100 Google.com results (in incognito mode) harvesting their URL for each of the following queries:

  • "C-Section"
  • "CS delivery"
  • "Surgical delivery"
  • "Abdominal delivery"
  • "Cesarean delivery"
  • "Caesarean delivery"
  • "Operative delivery"
  • Caesarean Cesarean

The different queries have been designed to address as much as possible "nuances" of the CS concept: from the more technical and medical ones ("Caesarean delivery", "Operative delivery") to the more ordinary ones ("C-Section", "CS delivery".

Rogers, Richard. "The Googlization Question, and the Inculpable Engine."

cleaning and tagging the webpages

We qualified and tagged all the webpages accordingly to their contents and excluding from the corpus pages not concerned with CS (i.e. PDF, Videos but also cesarean section on animals). In this way we obtained three main webpages groups:

  • a) Medical Information;
  • b) Contentious Information;
  • c) Experiential Information.

Crawling the hyperlinks

We followed the web corpus hyperlinks to discovering other relevant URLs. From this new web corpus have been eliminated all the websites considered too general (i.e facebook, twitter) and the non relevant ones (i.e maps.google, youtube). Furthermore, also the websites with an in-degree index smaller than 3 have been excluded. For this step we used HYPHE a software developed at m√©dialab | SciencesPo. Rogers, Richard. (visual) network) analysis

We examined through co-link / social networking analysis the network produced with the crawling. Furthermore, we analyzed visually the network using GEPHI, an open-source network analysis and visualization software package.
