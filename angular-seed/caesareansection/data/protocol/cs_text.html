scraping the text corpus

We scraped the text of each webpage contained in the web corpus built in the previous phase. The resulting documents kept the categorization of the original web pages and was therefore organized into three groups:

  • a) Medical Information;
  • b) Contentious Information;
  • c) Experiential Information.delivery"

In order to have as much clean content as possible we applied BOILERPIPE, an algorithm used to detect and remove the surplus "clutter" (boilerplate, templates) around the main textual content of a web page.

Kohlschütter, Christian, Peter Fankhauser, and Wolfgang Nejdl. "Boilerplate detection using shallow text features." Proceedings of the third ACM international conference on Web search and data mining. ACM, 2010.

n-gram extraction and tf/tf-idf analysis

We processed the text corpus using some natural language processing techniques as fast part-of-speech tagger (identification of nouns, adjectives, verbs, etc. in a sentence) and n-grams extractor. For each extracted entity we calculated its Term Frequency (TF) and its term frequency–inverse document frequency (TF-IDF), performed along the three categories of documents. We also excluded some overly generic entities and we used stop-word lists.
For this phase we used SVEN, a software developed at DensityDesign | Politecnico di Milano and médialab | SciencesPo, that implement Pattern a text mining library developed at CLiPS | University of Antwerp.

De Smedt, Tom, and Walter Daelemans. "Pattern for python." The Journal of Machine Learning Research 98888 (2012): 2063-2067.

visual relation n-grams/categories

The resulting n-grams have been visualized with a slopegraph model that allow to make several visual comparisons along the different categories of documents. All vertical lists in the diagram contain the same n-grams, thus highlighting the hierarchy of n-grams in the different categories and the specific number of occurrences associated to them, as well as the changing configuration when comparing TF and TFIDF n-grams.
